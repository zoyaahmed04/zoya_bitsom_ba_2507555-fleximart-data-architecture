## Overview
This part of the project explores the use of MongoDB as a NoSQL database
solution for handling a highly diverse and flexible product catalog for
FlexiMart.

## Objectives
- Analyze the limitations of relational databases for diverse product data
- Justify the use of MongoDB for flexible and scalable data storage
- Implement basic MongoDB operations using JSON documents

## Files in This Folder
- products_catalog.json: Sample product catalog data with flexible schema
- mongodb_operations.js: MongoDB operations including queries, updates, and aggregations
- nosql_analysis.md: Theory report explaining NoSQL justification and trade-offs

## Key Concepts Demonstrated
- Flexible document-based schema
- Embedded documents for customer reviews
- Aggregation operations in MongoDB
- Comparison between SQL and NoSQL databases
